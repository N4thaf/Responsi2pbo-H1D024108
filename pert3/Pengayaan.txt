Pertanyaan Refleksi
Setelah menyelesaikan kode, jawab pertanyaan berikut:

1. Apa fungsi keyword 'extends' dalam deklarasi class Java?
2. Jelaskan perbedaan antara Method Overloading (yang ada di latihan sebelumnya) dengan Method Overriding (yang ada di latihan ini)!
3. Apa kegunaan keyword 'super' di dalam constructor subclass (Manajer)?
4. Jika variabel di class Karyawan diubah access modifier-nya menjadi 'private', apa yang akan terjadi pada class Manajer? Bagaimana solusinya?
5. Mengapa kita perlu menggunakan Inheritance? Apa keuntungan utamanya dibandingkan membuat dua class yang terpisah sama sekali (misal class Karyawan dan class Manajer dibuat terpisah tanpa hubungan)?

Jawaban:

1. Keyword 'extends' digunakan untuk membuat hubungan inheritance (pewarisan) antara dua class, dimana subclass (child class) mewarisi semua atribut dan method dari superclass (parent class). Dengan extends, subclass dapat menggunakan kembali kode yang sudah ada di parent class dan menambahkan atau memodifikasi fitur sesuai kebutuhan.
2. Method Overloading adalah membuat beberapa method dengan nama yang sama tetapi parameter berbeda (jumlah atau tipe) dalam satu class yang sama. Method Overriding adalah mendefinisikan ulang method yang sudah ada di parent class dengan signature yang sama (nama, parameter, return type) di subclass untuk mengubah implementasinya.
3. Keyword 'super' di constructor subclass digunakan untuk memanggil constructor parent class agar atribut yang dimiliki parent class dapat diinisialisasi terlebih dahulu. Ini wajib dilakukan karena subclass mewarisi atribut parent, sehingga atribut tersebut harus diisi nilai sebelum subclass menambahkan inisialisasi atribut miliknya sendiri.
4. Jika variabel di class Karyawan diubah menjadi private, maka class Manajer tidak bisa mengakses variabel tersebut secara langsung. Solusinya adalah mengubah access modifier menjadi 'protected' (agar bisa diakses subclass) atau menyediakan getter/setter method untuk mengakses dan mengubah nilai variabel tersebut dari subclass.
5. Inheritance memungkinkan code reusability (penggunaan kembali kode), mengurangi duplikasi kode, memudahkan maintenance karena perubahan di parent class otomatis berlaku untuk semua subclass, membuat struktur kode lebih terorganisir dengan hubungan hierarki yang jelas, dan mendukung polimorfisme sehingga objek subclass dapat diperlakukan sebagai objek parent class.
